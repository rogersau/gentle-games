# Features Research: Sensory-Friendly Kids Apps

**Analysis Date:** 2026-03-03

## Current State

### Completed Features
- Memory matching game
- Bubble popping (sensory)
- Drawing canvas (creative)
- Glitter globe (motion-reactive)
- Balloon keepy-uppy (physics)
- Breathing exercises (calming)
- Pattern recognition
- Category matching

### Unfinished Features
- Number learning game
- Letter learning game
- Star collection game

## Table Stakes

These features are expected in sensory-friendly kids apps:

1. **Accessibility Controls** — Sound on/off, animation on/off, color themes
2. **Parental Controls** — Timer limits, content restrictions
3. **Offline Operation** — All games work without internet
4. **Gentle Onboarding** — No tutorials, immediate play
5. **Progress Persistence** — Save state locally

## Differentiators

Features that make this app special:

1. **Sensory Calibration** — Allow users to customize sensory experience intensity
2. **Calming Corner** — Dedicated space for breathing/relaxation activities
3. **Creative Tools** — Drawing, building, open-ended play
4. **Parent Dashboard** — Simple view of play time, favorite activities
5. **Customizable Activities** — Adjust difficulty, timing, stimuli

## Anti-Features

Things to deliberately NOT build:

| Feature | Reason |
|---------|--------|
| Leaderboards | Creates pressure, competition |
| Time Limits | Counter to calm, pressure-free principle |
| Social Sharing | Privacy concerns, not aligned with value |
| In-app Purchases | Breaks trust, adds complexity |
| Notifications | Potential sensory overload |
| Multiplayer | Adds complexity, not necessary |

## Dependencies

- Settings system → All features
- Parent timer → Timer feature
- i18n → All content
